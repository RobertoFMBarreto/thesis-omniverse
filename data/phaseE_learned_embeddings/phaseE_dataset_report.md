# Phase E.1 — Footprint SDF Dataset for Embedding Learning

> **This dataset supports learning geometric embeddings for affordance ranking; it does not train a robot controller.**

> Status: Phase E.1 (dataset generation only). No model has been trained from this data.

## Objective

Provide a procedurally-generated dataset of (piece footprint, cavity opening footprint, partial-insertion label) triples, rasterised at 128×128 with mask + signed distance field channels, suitable for training a learned geometric embedding (e.g. shallow CNN siamese encoder, see Doc 09).

## Representation choice

Each pair stores BOTH the binary mask and the signed distance field of piece and cavity at 128×128. The SDF is the primary candidate input for the embedding model; the mask is available for ablation and debug visualisation.

## SDF convention

- Positive **inside** the shape; negative **outside**; ~0 on the boundary.
- Computed in millimetres via `cv2.distanceTransform` on each side, then differenced.
- Clipped to ±20.0 mm.
- Quantised to `int8` via `sdf_int8 = round(sdf_mm × 6.3500)`.
- Dequantise: `sdf_mm = sdf_int8 / 6.3500`.

## Dataset source

Procedural shape generation reused from Phase D.7 (`scripts/generate_phaseD_3d_affordance_dataset.py`) via direct Python import. Same RNG seed, same families, same instances, same matching/mismatched cavity recipes. Labels generated by the same Phase D.7 partial-insertion rule; perception is deterministic and frozen.

## Label rule (inherited from Phase D.7)

```
lateral_ok = (outside_ratio_raw <= 0.05) AND
              (inside_ratio_raw  >= 0.8)

insertion_required_mm = max(
    5.0,
    0.25 * piece_height_mm
)

depth_ok = (cavity_depth_mm >= insertion_required_mm - 0.5) AND
            (cavity_depth_mm >= 5.0)

label = lateral_ok AND depth_ok
```

Constants are **fixed operating points**, not free tuning parameters.

## Limitations

- **Synthetic only**: labels generated by raster geometry; not physical insertion.
- **Convex prismatic only**: extrusion assumption inherited from Phase D.
- **No XY offset**: rotations only.
- **Perception frozen**: this dataset feeds learned embeddings, not learned perception.
- **No model trained from this data yet** (Phase E.1 is dataset generation).
- **MVP hold-in is a small minority** (4 pieces vs 100 procedural).

## Closing note

This dataset supports learning geometric embeddings for affordance ranking. It does **not** train a robot controller, **not** learn perception, **not** reconstruct full 3D, **not** model contact dynamics.
